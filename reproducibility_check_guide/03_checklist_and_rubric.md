# 交付前自查清单与评价标准

本清单用于交付前自查。每项先勾选完成情况，再根据文末等级标准判断整体是否达到交付要求。

## 1. 文献与数据准备检查

- [ ] 已记录原文献完整 APA 信息。
- [ ] 已保存或记录原文献 PDF 来源。
- [ ] 已记录数据来源链接，如 OSF、GitHub 或补充材料。
- [ ] 已检查是否提供原始数据。
- [ ] 已检查是否提供处理后数据。
- [ ] 已检查是否提供原分析代码。
- [ ] 已检查是否有 README、变量说明或数据字典。
- [ ] 已记录原文使用的软件、包、版本或分析方法。
- [ ] 已定位待复现结果在原文中的位置，如表号、图号、页码或段落。

## 2. 代码可运行性检查

- [ ] 已提供完整可运行代码，如 `.R` 或 `.Rmd`。
- [ ] 代码能从项目说明指定的位置开始运行。
- [ ] 数据读取路径清楚，不依赖个人电脑上的隐含路径。
- [ ] 代码包含必要注释，能说明主要步骤。
- [ ] 已记录使用的 R 版本和包版本，如 `sessionInfo()`。
- [ ] 若代码无法完整运行，已记录失败步骤和报错信息。
- [ ] 若对原代码做了修改，已说明修改内容和原因。

## 3. 数据预处理检查

- [ ] 已说明使用原始数据还是处理后数据。
- [ ] 若原文提供原始数据和处理方法，已优先从原始数据开始；若没有，已说明原因。
- [ ] 已记录缺失值处理方式。
- [ ] 已记录变量计算、转换或筛选方式。
- [ ] 已记录类别变量编码含义。
- [ ] 已列出最终纳入分析的变量。
- [ ] 已说明变量类型，如称名、顺序、等距、等比，连续或离散。
- [ ] 若修改样本，已说明理由、标准、步骤、原样本量和修改后样本量。

## 4. 描述性统计复现检查

- [ ] 已先检查描述性统计，再检查推断性统计。
- [ ] 已比较样本量 `N`。
- [ ] 已比较均值 `Mean`。
- [ ] 已比较标准差 `SD`。
- [ ] 原文献报告值和复现值已并列呈现。
- [ ] 每个可比较数值都已计算 PE / δ，或说明无法计算的原因。
- [ ] 每个可比较数值都已给出评级。
- [ ] 若描述性统计出现偏差，已说明是否影响后续推断性统计。

## 5. 推断性统计复现检查

- [ ] 已说明原文献使用的统计方法。
- [ ] 已尽量使用原文献的软件、包、方法或模型设定。
- [ ] 若方法或设定有改动，已说明原因。
- [ ] t 检验已比较 t 值、Cohen’s d 和 p 值。
- [ ] ANOVA 已比较 F 值、η² 或 partial η² 和 p 值。
- [ ] 相关分析已比较 r 值和 p 值。
- [ ] 回归分析已比较 t 值、R² 和 p 值。
- [ ] 卡方检验已比较 χ²、Cramér’s V 和 p 值。
- [ ] Bayes 分析已根据原文报告比较 BF、参数估计、CI / HDI 或后验评估指标。
- [ ] 每个可比较指标都已填写原文值、复现值、PE / δ、评级和备注。

## 6. PE / δ 计算检查

- [ ] 已使用公式 `PE = |原文献报告值 - 复现值| / |原文献报告值| × 100%`。
- [ ] 已统一使用 PE 或 δ 表示百分误差。
- [ ] 已按规则标记“完全一致”：PE = 0%。
- [ ] 已按规则标记“次要偏差 / 偏差较小”：0% < PE < 10%。
- [ ] 已按规则标记“主要偏差 / 偏差较大”：PE ≥ 10%。
- [ ] 已单独标记可由小数位数或四舍五入解释的舍入偏差。
- [ ] 数据缺失、代码缺失、说明不清或无法定位的结果已标记为“无法检验”。
- [ ] 原文报告值为 0 时，没有强行套用 PE 公式，而是单独说明。

## 7. 推论一致性检查

- [ ] 已确认原文是否指定显著性水平。
- [ ] 若原文未指定，已使用默认 `α = .05`。
- [ ] 已比较原文 p 值和复现 p 值是否在 α 边界同侧。
- [ ] 原文 p < .05 且复现 p < .05 时，已标记为一致。
- [ ] 原文 p ≥ .05 且复现 p ≥ .05 时，已标记为一致。
- [ ] 原文 p < .05 但复现 p ≥ .05 时，已标记为不一致。
- [ ] 原文 p ≥ .05 但复现 p < .05 时，已标记为不一致。
- [ ] 已汇总一致和不一致推论的数量与占比。

## 8. 创新方法检查（可选）

- [ ] 若使用创新方法，已说明该方法是什么。
- [ ] 若使用创新方法，已说明为什么该方法适合当前问题。
- [ ] 若使用创新方法，已说明相比原方法的可能优势或限制。
- [ ] 若使用创新方法，已报告新方法的结果。
- [ ] 若使用创新方法，已比较新方法推论与原文推论是否一致。
- [ ] 若未使用创新方法，报告中已明确说明未进行该部分分析。

## 9. 结果解释与讨论检查

- [ ] 已总结完全一致、偏差较小、偏差较大、舍入偏差和无法检验的结果数量。
- [ ] 已说明哪些结果无法复现或无法检验。
- [ ] 已说明无法复现或不一致的可能原因。
- [ ] 已区分原文献开放性问题、数据问题、代码问题、复现过程差异和其他因素。
- [ ] 已说明偏差是否影响原文研究假设。
- [ ] 已避免把无法检验的结果写成支持或不支持原结论。
- [ ] 已在讨论中说明本次复现的限制。

## 10. 最终文件交付检查

- [ ] 已提供 README 或项目说明。
- [ ] 已提供完整可运行代码。
- [ ] 已提供数据预处理代码。
- [ ] 已提供结果比较表。
- [ ] 已提供推论一致性表。
- [ ] 已提供可复现性汇总表。
- [ ] 已提供 R 版本和包版本信息。
- [ ] 已提供 error log 或问题记录。
- [ ] 已提供最终报告。
- [ ] 最终报告中的表格、代码输出和文字结论一致。
- [ ] 没有编造数据、代码、结果或原文没有提供的信息。

## 等级式评价标准

### 优秀

- 文献、数据、代码和结果位置记录清楚。
- 代码可以顺利运行，结果能追溯到原始或处理后数据。
- 描述性统计和推断性统计比较完整。
- PE / δ、评级和推论一致性判断准确。
- 无法复现的部分记录具体，不回避问题。
- 报告结构完整，表格清楚，讨论能解释偏差来源和影响。

### 合格

- 主要材料和结果位置基本记录清楚。
- 代码能运行主要分析，少量问题有说明。
- 完成了核心描述性统计和推断性统计比较。
- PE / δ 和推论一致性判断基本正确。
- 无法复现的部分有基本记录。
- 报告结构完整，但部分解释或表格仍可更清楚。

### 需修改

- 文献、数据或代码来源记录不清。
- 代码无法运行，且没有说明失败原因。
- 缺少核心描述性统计或推断性统计比较。
- PE / δ 计算、评级或推论一致性判断明显错误。
- 对无法复现的部分没有记录，或把无法检验写成确定结论。
- 报告、代码和表格之间存在明显不一致。
