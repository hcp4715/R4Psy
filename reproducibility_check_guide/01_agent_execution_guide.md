# Agent 执行指南：心理学量化研究计算可复现性检验

本指南供后续 Agent 执行课程中的分析 / 计算可复现性检验任务。执行时不得编造数据、代码或结果；遇到材料缺失或无法运行的情况，应记录事实并标记为“无法检验”。

## 1. 任务目标

Agent 的目标是使用原文献提供的数据、代码和方法，尽可能复现原文关键数值、图表和结论。

必做目标：

- 检查原文关键描述性统计结果是否可复现。
- 检查原文关键推断性统计结果是否可复现。
- 检查基于显著性水平的推论是否一致。
- 记录无法复现、无法比较或无法定位的结果及原因。
- 生成可运行代码、比较表、推论一致性表和最终报告。

可选但推荐：

- 在原文方法之外，使用新的 R 方法或更合适的分析思路重新检验原文推论。
- 如果小组条件允许，由成员先独立分析，再讨论形成汇总结果。

## 2. 输入材料检查

开始前必须检查并记录以下材料是否存在：

| 材料 | 必做检查 |
|---|---|
| 原文献 PDF | 是否能定位研究问题、样本、方法、结果和表图。 |
| 数据来源 | 是否有 OSF、GitHub、补充材料或其他公开链接。 |
| 原始数据 | 是否提供；若提供，优先从原始数据开始。 |
| 处理后数据 | 若无原始数据，检查是否至少有可分析数据。 |
| 原分析代码 | 是否公开；若公开，检查语言、包和运行入口。 |
| README 或数据说明 | 是否解释文件内容、变量含义和运行顺序。 |
| 待复现结果位置 | 记录原文中的表号、图号、页码、段落或假设编号。 |
| 软件与包版本 | 记录原文使用的软件、R 包、统计方法和模型设定。 |

若关键材料缺失，不得补造。应记录缺失内容、影响范围和后续是否仍可部分检验。

## 3. 总体工作流

按以下 9 步推进。带“可选”的步骤不是课程硬性要求，但原指南推荐进行。

1. 确定文献。
2. 确定待复现的研究假设和样本。
3. 数据预处理（可选但推荐；若从原始数据开始则通常需要）。
4. 描述性统计及比较。
5. 推断性统计及比较（使用原文献方法）。
6. 推断性统计及比较（使用新方法，可选）。
7. 记录信息并撰写报告。
8. 汇报与交流。
9. 汇总文件并上传。

后续步骤依赖前序信息，但不要求完全线性推进。若后续发现文献、假设、数据或代码不可行，应回到前一步修正并记录原因。

## 4. 文献选择规则

选择文献时必须考虑：

- 研究主题是否有兴趣和必要背景知识。
- 数据和代码是否公开可获取。
- 是否提供原始数据及处理流程；若没有，是否至少有处理后可分析数据。
- 原文分析方法是否能由小组掌握，或能通过快速学习掌握。
- 是否属于心理学各领域的人类心理与行为相关量化研究。
- 原文中是否至少包含一部分课程涉及的数据分析方法。

优先考虑但不强制：

- 主题包含不止一篇研究，或已有相关元分析。
- 文献此前被其他研究者成功或失败复现过。
- 原文使用 R 完成分析并提供代码。
- 文献来自指南列出的开放科学实践较友好的期刊，如 *Science*、*Nature*、*Science Advances*、*Nature Human Behavior*、*Nature Communications*、*Psychological Science*、*Cognition*、*Journal of Experimental Psychology: General*、*Advances in Methods and Practices in Psychological Science*、*Collabra: Psychology*。

## 5. 假设与数据集选择规则

选择待复现假设时，按以下顺序：

1. 优先复现文献中已经过可复现性检验的研究假设。
2. 若没有已检验假设，默认复现最后一个实验中的研究假设。
3. 若最后一个实验不可行，则从后向前选择较靠后的实验。

必须在代码文件和报告文件中记录：

- 最终选择的研究假设。
- 选择该假设的理由。
- 若没有按推荐顺序选择，说明原因。

选择数据集时：

- 如果原文提供原始数据和数据处理方法，应优先从原始数据开始，而不是直接使用处理后数据。
- 非必要不修改样本。
- 若必须修改样本，必须记录理由、标准、步骤、原样本量、修改后样本量，并按原指南要求报告修改后的效应量相关信息。

## 6. 数据预处理记录规则

若进行了数据预处理，必须在代码和报告中记录：

- 做了什么处理。
- 为什么做这些处理。
- 变量如何计算、转换或筛选。
- 缺失值如何处理。
- 类别变量如何编码。
- 最终纳入哪些变量。
- 每个变量是什么类型，例如称名、顺序、等距、等比，连续或离散。
- 类别变量的各数值分别代表什么类别。

代码中应保留清楚注释，让后续读者能追踪每一步如何从原始数据得到分析数据。

## 7. 描述性统计复现规则

必须先检查描述性统计，再检查推断性统计。原因是样本量、均值或标准差不一致，可能会影响后续推断结果的有效性。

至少比较：

- 样本量 `N`
- 均值 `Mean`
- 标准差 `SD`

每个可比较数值都必须并列呈现：

- 原文献报告值。
- 复现值。
- PE / δ。
- 评级。
- 必要备注。

若原文还报告了其他描述性结果，也可一并比较。

## 8. 推断性统计复现规则

使用原文献方法时，应尽量采用原文献的编程语言、软件包、分析方法和模型设定。若必须改动，必须说明改动内容和原因。

传统统计方法至少关注以下指标：

| 统计方法 | 统计量 | 效应量 | 显著性指标 |
|---|---|---|---|
| t 检验 | t 值 | Cohen’s d | p 值 |
| 方差分析 | F 值 | η² 或 partial η² | p 值 |
| 相关分析 | r 值 | Pearson’s r | p 值 |
| 回归分析 | t 值 | R² | p 值 |
| 卡方检验 | χ² 值 | Cramér’s V | p 值 |

Bayes 分析可关注：

| 统计方法 | 决策标准 | 参数估计 | 区间 | 后验评估 |
|---|---|---|---|---|
| 贝叶斯分析 | BF | β、Posterior Mean、Posterior SD | CI / HDI | ELPD、MAE、WAIC、LOO-CV |

每个可比较指标都应记录原文献报告值、复现值、PE / δ、评级和备注。

可选的新方法分析：

- 只在有理论或方法理由时进行。
- 可以使用与原文不同的 R 方法、模型或分析思路。
- 因分析思路改变，通常不直接比较具体数值；重点比较推论是否一致。
- 必须说明新方法为何可行，以及相比原方法可能有什么优势或限制。

## 9. PE / δ 计算规则

对每个可比较数值计算：

```text
PE = |原文献报告值 - 复现值| / |原文献报告值| × 100%
```

报告中可使用 PE 或 δ 表示该百分误差，但同一份报告中应保持一致。

评级规则：

| 评级 | 判断标准 |
|---|---|
| 完全一致 | PE = 0% |
| 次要偏差 / 偏差较小 | 0% < PE < 10% |
| 主要偏差 / 偏差较大 | PE ≥ 10% |
| 舍入偏差 | 差异可由小数位数或四舍五入解释 |
| 无法检验 | 数据缺失、代码缺失、说明不清或无法定位对应结果 |

注意：若原文献报告值为 0，PE 公式会出现除以 0 的问题。此时不要强行计算 PE，应记录为特殊情况，并用文字说明差异。

## 10. 推论一致性判断规则

默认显著性水平为 `α = .05`，除非原文献另有说明。

| 原文 p 值 | 复现 p 值 | 判断 |
|---|---|---|
| p < .05 | p < .05 | 一致 |
| p ≥ .05 | p ≥ .05 | 一致 |
| p < .05 | p ≥ .05 | 不一致 |
| p ≥ .05 | p < .05 | 不一致 |

推论数量 `N` 指在复现分析中对效应做出统计推断的次数。比如一个 t 检验通常为 1 次推论；一个 2 × 2 方差分析加简单效应分析可能包含多个推论。

## 11. 无法复现时的处理规则

不得编造结果。遇到无法复现或无法检验时，必须记录：

- 缺失了什么数据、代码或说明。
- 失败发生在哪一步。
- 报错信息或异常现象。
- 已尝试的修复方式。
- 是否影响后续分析。
- 该结果是否应标记为“无法检验”。

如果只能复现部分结果，应继续完成可复现的部分，并清楚标注哪些结果无法检验。

## 12. 最终交付物清单

最终至少应包含：

- README 或项目说明。
- 完整可运行代码，例如 `.R` 或 `.Rmd`。
- 数据预处理代码。
- 结果比较表。
- 推论一致性表。
- 可复现性汇总表。
- `sessionInfo()` 或包版本信息。
- error log 或问题记录。
- 最终报告。

交付前使用 `03_checklist_and_rubric.md` 自查。若发现结果、代码和报告之间不一致，应先修正再交付。
