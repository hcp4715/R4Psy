Research logic

从制度摩擦到因果检验

点击左侧步骤，可以看到作者为什么按这个顺序推进：先定位不平等发生在哪个环节，再检验哪类干预能改变行为。

制度背景

法国 0-3 岁早期托育包括 daycare centres 和 childminders。它不是自动获得的福利，而是需要家庭理解选项、判断成本、提前申请并等待分配。名额供给不足和规则不透明会让申请之后的 access 仍然存在摩擦。

因此，这篇文章不是只问“家庭有没有需求”，而是把托育进入过程拆成 intention、application 和 access 三段。

Estimation strategy

统计分析如何层层推进

这部分对应正文复现脚本的核心分析顺序。重点是每一步回答的统计问题，而不是代码细节。

Control-group gaps

只在 control group 内比较 intention、application 和 access，定位不平等在哪个阶段扩大。

ITT

用随机分配估计 treatment assignment 的平均因果效应，比较 T1-C、T2-C 和 T2-T1。

ATT / LATE

用 T2 assignment 工具化实际 support take-up，估计 compliers 的支持效果。

Heterogeneity

按 SES、移民背景和机制 proxy 分组，判断干预是否缩小不平等以及可能通过什么机制发挥作用。

Conclusion

这条证据链最终说明什么

结论需要同时保留正向效果和边界条件：支持能提高申请，但不能单独解决 access。

信息本身不足够

Information-only treatment 的平均效果很小，说明把信息做得更清楚、更及时并不能自动让家庭完成申请。

行政支持改变申请

Information + support treatment 显著提高申请率，尤其帮助低 SES、移民背景、缺少既有经验或周围使用托育较少的家庭。

access 仍有后续障碍

申请增加没有完全转化为获得名额。若政策目标是缩小最终 access inequality，还需要处理供给、分配和机构筛选环节。

Method Primer

为什么 RCT 主分析通常先看 ITT

这一节是给第一次读 RCT 结果的人看的：先把“随机分配产生了什么因果比较”讲清楚，再解释为什么文中还会报告 ATT。

ITT 的核心问题

ITT, intention-to-treat, 估计的是“被随机分配到某个处理组”这件事的效果。它不问一个家庭最后有没有认真看完视频、有没有接到电话、有没有完整配合申请支持，而是按照最初随机分配的组别来比较结果。

这篇文章里，ITT 最关心的是一个政策式问题：如果把一批家庭随机分配到 information-only 或 information + support，会不会提高申请和获得托育的概率？这种问题很适合政策评估，因为现实中的政策推广本来就会遇到未接触、未使用、未完全配合等情况。

保留随机化 估计“被提供干预”的效果 主线结论最稳健
1

先固定随机分配组别

控制组没有收到研究团队额外提供的视频信息和电话申请支持；T1 收到短信视频和提醒；T2 在 T1 的信息基础上再获得电话申请支持。

2

不按实际参与程度重新分组

ITT 的关键是“as randomized”。即使有人没有完整接触干预，也仍然留在原先被分配的组里，这样才不破坏随机化。

3

比较三组政策暴露

文中报告 T1-C、T2-C、T2-T1。前两个比较处理组相对控制组的效果，第三个比较“在已有信息基础上再加行政支持”的边际效果。

4

把结果解释为百分点变化

例如 T2-C 在 daycare application 上的正系数，表示被分配到 information + support 组让申请 daycare 的概率相对控制组提高了多少个百分点。

ITT 问的是 assigned effect 被分配到某个干预组，相比被分配到对照组，结果是否改变。
First stage 问的是 take-up 被分配到支持组，是否真的提高了实际收到行政支持的概率。
ATT/LATE 问的是 treated effect 对实际因为随机分配而收到支持的家庭，支持本身带来多大效果。
这篇文章具体怎样估计 ITT

复现脚本中，主 ITT 使用 R 的 fixest::feols 做线性概率模型。结果变量包括 early childcare application、daycare application、early childcare access 和 daycare access。模型把三类比较堆叠起来估计：information-only vs control、information + support vs control、information + support vs information-only。

估计时加入随机分层相关的固定效应，按 StrataWave 聚类标准误，并使用响应概率相关权重。多重检验部分用 multcomp 做 Westfall 调整，表格整理用 modelsummary。这些设置的目的不是替代随机化，而是让估计更贴合原始实验设计和正文表格报告方式。

为什么 ITT 是这篇文章的主线，而不是只看真正接到电话的人

如果只比较“真正接到电话支持的人”和“没有接到的人”，这个比较就不再是随机的。能接通电话、愿意沟通、已经更积极准备申请的家庭，本来就可能和其他家庭不同。这样得到的差异可能混合了家庭本身的主动性、可联系性和资源差异。

ITT 避开了这个问题。它只利用最初的随机分配，所以可以更直接地说：把家庭分配到某个政策暴露下，会不会改变后续申请和 access。这也是 RCT 学习里最重要的一点：先保护随机化，再解释因果效果。

ATT 在这里补充回答了什么

ATT, average treatment effect on the treated, 在这篇文章中对应的是实际收到行政支持的家庭的效果。由于不是所有被分配到 T2 的家庭都一定实际收到或完成电话支持，作者进一步用随机分配作为工具变量，估计实际支持对 compliers 的影响。复现脚本里这个部分用的是 fixest 的 IV 写法，函数名也保留了原代码里的 LATE 思路。

直觉上，ITT 是“发出这个政策包以后，平均带来什么变化”；ATT/LATE 是“在那些因为被分配到支持组而实际接受支持的家庭中，支持本身有多大作用”。当实际接受率不是 100% 时，ATT/LATE 往往会比 ITT 更大，因为 ITT 被没有实际接触支持的人稀释了。

读这篇文章时怎样把 ITT 和 ATT 放在一起理解

先看 ITT，因为它最接近可推广政策问题：研究团队把信息和支持提供出去以后，是否真的改变了申请行为。正文结论中的“information-only 基本不足、information + support 能提高申请”主要就是从这个视角成立的。

再看 ATT，是为了理解实际支持的强度和机制。如果 ITT 不大但 first stage 明确，ATT 可能显示实际接受支持的人受益更明显；如果 ITT 和 ATT 都集中在低 SES、移民背景或缺少既有经验的家庭，就更支持“行政负担和信息成本阻碍弱势家庭把意愿转化为申请”的解释。

第一次做 RCT 复现时最容易误解的地方

第一，不要把 ITT 理解成“真正使用干预的效果”。ITT 是分配效果，不是实际参与效果。第二，不要因为有人没有接受干预就把他从处理组删掉，这会破坏随机化。第三，控制变量和固定效应是为了提高精度和对应实验设计，不是因为没有它们就不能识别因果效果。第四，显著提高 application 不等于显著提高 access，因为 access 还受到名额供给、机构分配和后续筛选影响。
